# My interview questions
Almost all questions are "opened" - like a start point for further discussion. For example, the question:\
What is a byte?\
Here we can deep dive into history of the computer science, talk about different architectures, the candidate can show his ken and erudition. Usually through 2-3 minutes you can understand the knowlidge level of the candidate in a question.

## Theoretical questions
What is a byte?

We write a program as a text but the processor does not understand the text and can’t run it in the text format. What does happen and how does our program is transformed to the processor can run it?
